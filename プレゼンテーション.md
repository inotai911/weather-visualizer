# 🌤️ 気象情報可視化システム
## Weather Visualization System

世界中の気象データをリアルタイムで可視化

---

## 📊 プロジェクト概要

### 目的
世界中の気象データを**簡単に**取得・可視化し、
誰でも天気情報を分析できるWebアプリケーション

### 主な特徴
- 🌍 世界中の都市に対応
- 📈 美しいグラフ表示
- 🗺️ インタラクティブな地図
- 📱 モダンなレスポンシブUI

---

## 🎯 主な機能

### 1. データ取得
- Open Meteo APIで無料の気象データ
- 都市名入力だけで世界中にアクセス
- 7日間の時間ごとの詳細予報

### 2. 可視化
- Chart.jsによる5種類のグラフ
  - 気温 / 湿度 / 降水量 / 風速 / 気圧

---

## 🗺️ 地図機能

### Leaflet.js による対話型地図
- 選択した地域へスムーズアニメーション
- マーカーでの位置表示
- 初期表示は世界地図

```javascript
map.flyTo([lat, lon], 10, { duration: 1.5 });
```

---

## 📊 統計サマリー

### リアルタイム統計情報
- ✅ 平均気温
- ✅ 最高気温 / 最低気温
- ✅ 平均湿度
- ✅ 総降水量
- ✅ 平均風速

すべてカード形式で見やすく表示

## 💾 データベース設計

### 3つのテーブル

1. **meteo_data** - 気象データ
   - 温度、湿度、降水量、風速、気圧など

2. **user_log** - 操作ログ
   - すべてのユーザー操作を記録

3. **locations** - 地域マスタ
   - 登録済み地域の座標管理

## 🔌 API エンドポイント

| メソッド | パス | 説明 |
|---------|------|------|
| GET | `/api/locations` | 地域一覧 |
| POST | `/api/weather/fetch` | データ取得 |
| GET | `/api/weather` | データ表示 |
| GET | `/api/weather/summary` | サマリー |
| GET | `/api/logs` | ログ取得 |

**RESTful設計で拡張性を確保**

## 💡 設計のポイント

### なぜFlask？
- 軽量で学習コストが低い
- Pythonの豊富なライブラリ活用
- 迅速なプロトタイピング

### なぜSQLite？
- セットアップ不要
- 単一ファイルで管理容易
- 小〜中規模に最適

### なぜChart.js？
- 美しいアニメーション
- レスポンシブ対応
- カスタマイズ性が高い
